Commandes GIT

Installation de Git et Configuration

Pour faire l'installation de Git, vous devez exécuter les commandes suivantes :

sudo apt install git

Pour configurer les préférences de l’utilisateur, utilisez la commande suivante :

git config --global user.email "votre.email@example.com"

Git init

Cette commande est utilisée pour créer un nouveau dépôt GIT :

git init

Git add

La commande git add peut être utilisée pour ajouter des fichiers à l’index :

git add temp.txt

Git clone

La commande git clone est utilisée pour la vérification des dépôts :

git clone alex@93.188.160.58:/chemin/vers/dépôt

Pour créer une copie de travail d’un dépôt local :

git clone /chemin/vers/dépôt

Git commit

La commande git commit permet de valider les modifications apportées au HEAD :

git commit -m "Description du commit"

Git status

La commande git status affiche la liste des fichiers modifiés :

git status

Git reset

Pour réinitialiser l’index et le répertoire de travail à l’état du dernier commit :

git reset --hard HEAD

Git rm

Git rm peut être utilisé pour supprimer des fichiers de l’index et du répertoire de travail :

git rm nomfichier.txt

Git stash

La commande git stash aide à enregistrer les changements temporairement :

git stash

Git show

Pour afficher des informations sur tout fichier git :

git show

Git fetch

Git fetch permet d’extraire tous les fichiers du dépôt distant :

git fetch origin

Git grep

Git grep permet de rechercher des expressions dans les arbres de contenu :

git grep "www.hostinger.com"

Git rebase

La commande git rebase est utilisée pour la réapplication des commits sur une autre branche :

git rebase master
